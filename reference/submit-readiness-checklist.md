# Submit-Readiness Checklist

The last gate before you hit submit. Gate 1 checked the build could run; this checks the launch is actually shippable. Every box ticks, or you're not done.

## The build
- [ ] Repo is **public** and the link resolves in an incognito window.
- [ ] A **worked sample is committed** — a stranger sees it work without running anything.
- [ ] Only **shareable data** is committed: no private CRM rows, no client names, no secrets. Obscured demo data only.
- [ ] `HANDOFF.md` and any build-coordination files are **gitignored** — not in the public tree.

## The README
- [ ] Passes the **5-minute stranger test** cold — someone with no context gets value in five minutes.
- [ ] Setup steps are numbered and copy-pasteable.
- [ ] Leads with the hook, not the methodology.

## The launch materials
- [ ] Every link in the blurb / landing / submission **resolves** — no 404s, no localhost.
- [ ] The landing hub is live, and the video and deck it points to actually load.
- [ ] The submission entry leads with the chosen hook (Gate 3).

## The judge's path
- [ ] A judge can verify the headline claim **cold** — via the judge guide's falsifiable prompts or the committed sample. (See `reference/judging-criteria.md`.)

If a box can't be ticked, that's the next task — not a launch.
