# Template — 60-Second Script

Two parts in one file: the **shot plan** (what's on screen, with timing) and the **spoken VO script** (the words alone, to record). They're kept separate on purpose — the VO block at the bottom is what the builder reads into the mic, and nothing else.

## Part 1 — Shot plan
Beats: 0–10s hook, 10–45s it working on one real input, 45–60s where to get it. Show a decision being made, not a feature tour.

| Time | On screen | VO line |
|---|---|---|
| 0:00–0:10 | [the hook — duel split, or the problem stated] | [hook line] |
| 0:10–0:25 | [one real input appears] | [what we're feeding it] |
| 0:25–0:45 | [the operator deciding — gates, verdict] | [the decision, in plain words] |
| 0:45–0:60 | [the name + repo URL / CTA] | [where to get it] |

## Part 2 — Spoken VO script (record this, verbatim)
First person, specific, fragments fine, no hype. Measured pace; ~150 words ≈ 60 seconds. No stage directions inside the block.

```
[Hook line — open mid-thought, not "Today I'm excited to show…"]

[The setup — one real input, named concretely.]

[The decision — the thing it does that a generic AI won't. Show the refusal or the routing.]

[The close — the name, where to get it. Short. No summary.]
```

## Bar
- [ ] Shows a decision, not a feature list.
- [ ] VO block is clean — record-ready, no stage directions inside it.
- [ ] ~150 words. Lands the hook in the first ten seconds.
