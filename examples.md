# Examples — Shipper

Four decisions in action.

## 1. Standard — solid runway, lived, side-by-side-able
**Input:** "Operator that triages prospects for my game-audio studio. Half a day before the deadline. I run BD for a real studio. It visibly refuses to profile when a generic AI would."
**Gate 1:** worked sample committed, README runs cold, five files present → PASS.
**Gate 2:** descend the leverage order — blurb + README, then the landing hub, then the 60-second script. Stop where the runway runs out.
**Gate 3:** side-by-side-able → lead with the duel demo.
**Output:** a launch plan (blurb, landing, short — in leverage order), the duel framing drafted, and a note that the lived-BD angle reinforces the founder story in the blurb.

## 2. Edge — wants a landing page, but no worked sample
**Input:** "Can you draft me a landing page? Build's basically done, I'll add an example later."
**The trap:** obliging and drafting the landing page.
**Shipper's rule:** Gate 1 blocks before Gate 2 is reached. A landing page pointing at a build with nothing to show converts no one and erodes trust.
**Verdict: BLOCK.** "Commit one real run first — that's the highest-leverage 20 minutes you have. The landing page is drafted the moment there's something for it to point at."

## 3. Edge — strong build, researched persona
**Input:** "Operator for wedding-photography client intake. I'm not a photographer — I researched the workflow. Plenty of runway."
**The trap:** descending the full leverage stack and polishing a persona build.
**Shipper's rule:** Gate 4 flags the persona before effort is spent. The winning builds are lived; distribution amplifies a real founder, it doesn't manufacture one.
**Verdict: FLAG, then route.** "This will read as researched, which caps it. Options: reframe to a workflow you've actually run, or own it explicitly ('built from N interviews, not lived'). If you ship as-is, lead with the system design, not a founder story you don't have."

## 4. The hook loop — no obvious hook, so Shipper proposes (and rejects its own first try)
**Input:** "Operator that turns my meeting notes into action items. Solid build, runs clean. Built it for my own team." No duel (every notes AI formats notes), no sharp refusal named, lived but unremarkable.
**Gate 3 detects:** nothing strong. A generic AI formats notes fine — no visible split — and there's no named refusal yet.
**Shipper proposes, then rejects itself:** First try — "The smart way to handle meeting notes." Self-check trips weak-hook rule 1 (category-swappable) and rule 2 (claims, doesn't show). Rejected, rules named.
**Re-proposes:** "Most notes AIs hand you a tidy summary. This one refuses to — it only emits action items with an owner, or it asks who owns it." Names a refusal, shows behavior, survives the rules.
**Verdict: PAUSE.** Surfaces the re-proposed hook for a yes/no — Shipper's one sanctioned pause — and holds the blurb, landing hero, and script until it's approved.
